# Muslingo Audit 2026-07-10

## Checked

- Login: guest entry removed from the login UI; old saved guest profile is not restored automatically.
- Registration: local PocketBase registration opens the main learning screen.
- Quran lesson audio: Quran listening steps now point to exact global ayah numbers instead of synthesized/reused text audio.
- Quran tab: 114 surahs / 6236 ayahs screen opens; Al-Faatihah opens; ayah audio sources are exact per global ayah number.
- Quran chapter flow: every surah screen now has a top full-surah listening row that plays all ayahs sequentially, before the individual ayah cards.
- Quran offline reading: the Quran screen now has a full offline download control that caches all 114 surahs for offline reading with progress.
- Quran full-surah audio: full-surah listening now uses one complete surah mp3 file, so it no longer switches between separate ayah files during normal playback.
- Quran audio fallback: if a complete surah mp3 cannot be opened, the app falls back to the old per-ayah queue instead of failing silently.
- Quran full text: every surah screen now opens a full-text sheet with Arabic and Russian modes.
- Quran web audio: Quran audio launch was moved to the raw pointer-down path and a single `playUrl` call so a real browser click starts playback before Chrome loses user activation.
- Auth errors: duplicate account errors no longer expose PocketBase `Value must be unique`; the UI now tells the user that the email already has an account and returns them to login mode.
- Lessons mode switch: Lessons now has a top switch between Surah study and Arabic language study.
- Arabic onboarding: first Arabic-mode entry asks for native language once: Russian, Kazakh, or Uzbek; the choice is saved in local preferences and shown in the switch.
- Arabic course: added a gamified Arabic-language path with letters, short vowels, syllables, questions, and speech steps.
- Surah course structure: Al-Faatihah is split into smaller lessons that cover meaning, context, ayahs, checks, and pronunciation instead of one oversized lesson.
- Mistake loop: wrong question steps are collected and repeated before the lesson can finish.
- Speech workflow: speaking steps now keep the Continue button disabled until speech is recognized and compared with the target phrase; the UI shows recognition text and match percent.
- Energy: completed lessons now award energy in addition to XP, and the top stats row displays energy as a real tracked value.
- Surah rewards: Quran lessons now end with three sequential reward boxes; the user must open all boxes before continuing.
- Haptics: added tactile feedback to mode switches, lesson navigation, answer checks, speech success, native-language selection, and reward boxes where supported by the device.
- Emoji cleanup: visible emoji usage and old `iconEmoji` model fields were replaced with Material icons across lessons, profile, streak, achievements, leaderboard, help, and review screens.
- Core navigation: Lessons, Quran, Rules, League, Profile tabs respond from the main screen.
- Profile actions: Settings and Muslingo+ open real screens.
- Static button scan: no empty `onPressed` / `onTap`, TODO placeholders, or guest login handler in the login screen.

## Remaining Risks

- In-app browser automation clicks are still reported by Chrome as not user-initiated for `HTMLAudioElement.play()`, so the final audible Quran playback check must be done by a real click in the open preview window. The code path was adjusted for real pointer-down clicks and the audio endpoints return valid mp3.
- Browser automation cannot accept microphone permission on the user's behalf. A real device/browser permission click is still required for the final audible speech-recognition test.
- Flutter web reports a fallback-font warning for some characters. Arabic is visible, but a dedicated Noto/Amiri fallback pass should be done for cleaner text rendering.
- Fully offline audio for all 114 surahs is not bundled into the app yet. The app now streams each surah as one file without ayah gaps; full offline audio should be added as an optional large download because Al-Baqarah alone is about 115 MB.
- Destructive account deletion was not clicked during audit.
